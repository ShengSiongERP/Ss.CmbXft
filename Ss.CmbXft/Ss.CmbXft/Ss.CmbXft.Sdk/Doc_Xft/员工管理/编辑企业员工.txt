接口描述
本接口是花名册员工编辑接口，支持编辑薪福通-员工管理-花名册中的员工信息。通过对象化的入参降低调用方的使用难度，参数字段采用具有语义化的驼峰命名，降低调用方的阅读理解成本。
使用该接口时，最大支持一次传入1000笔员工信息数据。
调用方式
METHOD: POST
CONTENT-TYPE: application/json

错误码说明
除HTTP返回码外，具体的业务异常错误码通常指的是返回报文中错误码字段的内容，常见通用错误码及解决方案请参考通用错误码。
错误码	错误信息
errorMessage	新增接口中的错误新增会统一获取后在返回结果中展示
本文档对应的创建企业员工接口，会将输入参数中员工信息的处理结果通过响应报文中body的errorMessage返回。
当errorMessage为""表示员工正常通过校验进行花名册。例如：

{  
    "returnCode": "SUC0000",  
    "errorMsg": null,  
    "body": [  
        {  
            "stfSeq": "0000000001",  
            "certificateType": "O",  
            "certificateNumber": "350721191111110403",  
            "errorMessage": ""  
        }  
    ]  
}  
 

当errorMessage不为""表示员工被校验拦截，需要根据errorMessage的提示内容对输入参数中的员工信息进行调整。例如：

{  
    "returnCode": "SUC0000",  
    "errorMsg": null,  
    "body": [  
        {  
            "stfSeq": "0000000001",  
            "certificateType": "O",  
            "certificateNumber": "350721191111110403",  
            "errorMessage": "员工号：与员工花名册的员工的员工号重复，请修改;部门：不存在"  
        }  
    ]  
}  


常见的提示信息与解决方案如下：
提示信息	解决方案
员工号：与同批次员工的员工号重复，请修改	入参报文中员工信息之间员工工号存在重复，请接口调用方进行调整，避免重复
员工号：与员工花名册的员工的员工号重复，请修改	入参报文中员工信息的员工工号与员工管理-花名册中的员工的员工工号重复，请接口调用方进行调整，避免重复
手机号：与同批的在职员工手机号重复，请修改	入参报文中员工信息之间手机号存在重复，请接口调用方进行调整，避免重复
手机号：与员工花名册的在职员工的手机号重复，请修改	入参报文中员工信息的手机号与员工管理-花名册中的员工的手机号重复，请接口调用方进行调整，避免重复
证件号码:本次处理员工之间存在重复,请修改！	入参报文中员工信息之间证件信息(证件类型+证件号码)存在重复，请接口调用方进行调整，避免重复
证件号码：与花名册员工的证件号码重复，请修改！	入参报文中员工信息的证件信息(证件类型+证件号码)与员工管理-花名册中的员工的证件信息(证件类型+证件号码)重复，请接口调用方进行调整，避免重复
汇报上级：花名册不存在对应人员，请新增人员	检查入参报文中员工信息的汇报上级属性，不能传递员工管理-花名册中不存在员工的员工序号
部门：不存在	检查入参报文中员工信息的部门属性，不能传递不存在的部门id
证件信息：该员工已实名认证，不支持更新，请通知员工前往【个人账号】修改或联系客服处理	已实名认证员工不能修改证件信息
手机号：该员工已实名认证，不支持更新，请通知员工前往【个人账号】修改或联系客服处理	已实名认证员工不能修改手机号
姓名：该员工已实名认证，不支持更新，请通知员工前往【个人账号】修改或联系客服处理	已实名认证员工不能修改姓名
仅支持通过任职管理修改字段信息，如需直接修改，可前往【员工管理-基础设置】打开【任职信息轻管理模式】开关	请前往接口调用方【员工管理-基础设置】，判断是打开【任职信息轻管理模式】开关还是调整入参
当前修改员工不存在,请检查员工序号是否传递正确	请调用方检查入参中员工信息的员工序号
员工状态:已离职状态暂不支持批量修改，请通过新增导入或在员工详情页修改	已离职员工的员工状态不支持修改
实际离职日期：在职员工不能维护实际离职日期	请接口调用方检查入参，在职员工不能维护实际离职日期
